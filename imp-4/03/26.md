## Phase 1: API Presentation Layer Upgrade

### Problem
The existing API endpoints returned inconsistent, raw data formats:
- No unified response envelope
- Raw pipe-delimited descriptions exposed to clients
- No relevance ranking on search
- Impact data flat (no direct/indirect separation)
- Signal phrases returned raw (e.g., `[STRONG] crisis (occurrencesĂ—4, sourcesĂ—3)`)
- No metadata summary (confidence, source count, last updated)

### Changes Made

**File: `api/routers/intelligence.py`** â€” Complete rewrite (354 insertions, 87 deletions)

#### 1. Unified Response Format
All read endpoints now return the same envelope:
```json
{
  "event": {},
  "timeline": {},
  "impact": {},
  "signals": {},
  "metadata": {
    "confidence_summary": { "min": 0.25, "max": 0.70, "avg": 0.48, "data_points": 2 },
    "data_sources": 1,
    "last_updated": "2026-04-01T04:00:00+00:00"
  }
}
```

#### 2. New Helper Functions Added
- `_clean_description()` â€” Splits pipe-delimited titles, joins with semicolons, truncates at word boundary (200 chars max)
- `_parse_impact_entries()` â€” Parses `DIRECT:`, `INDIRECT:`, `SECTOR:` prefixed entries into grouped dicts
- `_parse_signal_phrase()` â€” Extracts strength label (WEAK/MEDIUM/STRONG) and keyword from structured phrase strings
- `_build_event_payload()` â€” Constructs the event section with human-readable `event_type_label` and `impact_classification`
- `_build_timeline_payload()` â€” Builds timeline with batch-loaded causal connections per entry (top 5 by confidence)
- `_build_impact_payload()` â€” Groups impact into `short_term` / `long_term` Ă— `direct_winners` / `indirect_winners` / `direct_losers` / `indirect_losers` + `sector_impacts`
- `_build_signals_payload()` â€” Parses signal phrases into clean structured output with strength, keyword, entity, source_count
- `_build_metadata()` â€” Aggregates confidence scores across all layers (event + impact + signals), counts distinct data sources via cluster JOIN
- `_build_unified_response()` â€” Master builder composing the full envelope

#### 3. Improved `/event` Search
- **Before**: Searched only `entity`, `event_type`, `description`
- **After**: Also searches `EventCluster.main_topic` via `outerjoin`
- **Relevance ranking**:
  - Exact entity match: +10 points
  - Partial entity match: +5 points
  - Description match: +2 points
  - Recency bonus: up to +3 points (decays over 10 days)
  - Anchor bonus: +1 point
- Added configurable `limit` parameter (1-50, default 10)

#### 4. Improved `/timeline/{id}`
- Connections now show full node context (`entity`, `event_type`, `description`) not just IDs
- Split into `causes` (outgoing) and `caused_by` (incoming) arrays and they are sorted by confidence

#### 5. New Model Imports
Added `CleanedNews`, `ClusterNewsMap`, `EventCluster`, `RawNews` for the metadata source count query.

#### 6. Lookup Tables
- `_EVENT_TYPE_LABELS` â€” Maps internal types (`trigger`, `causal`, `reaction`, `update`) to human-readable labels
- `_IMPACT_TYPE_LABELS` â€” Maps impact types (`supply`, `demand`, `policy`, `financial`, `geopolitical`, `technology`) to labels

### Bugs Found & Fixed

#### Bug 1: Missing CORS Middleware
- **File**: `api/main.py`
- **Symptom**: Frontend cross-origin requests would be blocked
- **Fix**: Added `CORSMiddleware` with `allow_origins=["*"]`, `allow_methods=["*"]`, `allow_headers=["*"]`

#### Bug 2: Duplicate RSS Ingestion Crash
- **File**: `ingestion/service.py`
- **Symptom**: `sqlite3.IntegrityError: UNIQUE constraint failed: raw_news.unique_id`
- **Root cause**: RSS feeds return multiple items with identical `title + source + None timestamp`, generating the same SHA256 hash. Items were checked against DB but not against each other within the same batch.
- **Fix**: Added `seen_in_batch: set[str]` that tracks all unique_ids within a single `insert_raw_news()` call. Duplicates within the batch are skipped before attempting DB insert.

#### Bug 3: Timezone-Naive Datetime Arithmetic
- **File**: Created `utils/datetime_helpers.py`
- **Files affected**: `timeline/service.py` (2 locations), `signal_detection/service.py` (2 locations), `api/routers/intelligence.py` (1 location)
- **Symptom**: `TypeError: can't subtract offset-naive and offset-aware datetimes`
- **Root cause**: SQLite returns datetimes without timezone info (naive), but the code uses `datetime.now(timezone.utc)` which creates timezone-aware datetimes. Subtracting the two fails.
- **Fix**: Created `ensure_utc(dt)` helper that attaches `timezone.utc` to naive datetimes. Applied at all 5 subtraction points:
  - `timeline/service.py:160` â€” `_temporal_signal()` time gap calculation
  - `timeline/service.py:348` â€” Anchor freshness scoring
  - `signal_detection/service.py:131` â€” Signal time span calculation
  - `signal_detection/service.py:136` â€” Signal staleness decay
  - `api/routers/intelligence.py:401` â€” Search relevance recency bonus

#### Bug 4: PostgreSQL Required for Local Development
- **File**: `database/config.py`, `database/session.py`, `.env`
- **Symptom**: Backend fails to start without PostgreSQL running (`connection refused port 5432`)
- **Fix**:
  - Added `DATABASE_URL` env var with priority over individual `DB_HOST/PORT/NAME/USER/PASSWORD` fields
  - Made all PostgreSQL fields optional with defaults
  - SQLite-specific engine config: `check_same_thread=False`, no `pool_pre_ping`
  - `.env` now uses `DATABASE_URL=sqlite:///news_intelligence.db` for local dev

## Phase 4: API Search Upgrade (v2.0) & RSS Expansion

### RSS Expansion
- Expanded `.env` `RSS_FEEDS` to include 10 global and Indian sources (BBC World, NYT World, Al Jazeera, The Hindu, Indian Express, Times of India, Livemint, Reuters World, Reuters Business, BBC Business).
- Note: Reuters feeds occasionally throw `[Errno 11001] getaddrinfo failed` due to strict bot-protection/DNS. The pipeline handles this gracefully, skipping the dead feed and processing the others.

### Advanced Multi-Keyword & Synergy Search
The `/event` search logic was entirely overhauled to powerfully support multi-word natural queries (e.g. "UAE wants to help US force Hormuz open").

**Files modified:** `api/routers/intelligence.py`

#### Key Algorithmic Improvements:
1. **Query Tokenization (`_tokenize_query`)**: Splits input, strips punctuation/spacing.
2. **Generic Word Penalty**: Moved verbs like "us", "help", "want", "say" from `_STOPWORDS` into `_LOW_WEIGHT_WORDS`. Instead of ignoring them, it safely penalizes their match value down to +0.5.
3. **Synonym Limitation Mapping**: Automatically expands tokens ("uae" â†’ "united", "arab", "emirates"). Mapped specifically *only* to the `entity` field in SQL to prevent false positives in descriptions.
4. **Entity Priority Boost**: Entity field hits weigh a dominant `+10.0` points, overpowering topic hits `+5.0` or description hits `+2.0`.
5. **Synergy Multiplier**: If an article hits multiple unique, non-generic tokens in the user's sentence, it gains an exponential synergy multiplier `matched_kws * 2.0`.
6. **Minimum Strong Match Filter (Noise Reduction)**: If a search query only generates matches via generic words ("help"), and records `strong_matches == 0`, the total score receives an extreme 90% penalty (`score *= 0.1`).

These changes ensure the system seamlessly returns ranked answers for copy-pasted headlines or nuanced topic searches.
